## Get started with Nitric in minutes

Deploy your first application on the Nitric platform in just a few steps.

<Note>
  **Before you begin:** Make sure you have Nitric CLI installed. See our [Installation Guide](/docs/cli/installation) for platform-specific instructions.
</Note>

<AccordionGroup>
  <Accordion title="Setup" icon="rocket">
    **Create and initialize your project:**

    ```bash
    # Create new project from template
    nitric new my-first-app
    ```

    ```bash
    Welcome to Nitric, this command will help you create a project from a template.
    
    Project name: my-first-app
    
    ? Template: 
      > acme/api-starter - REST API template
        acme/web-app - Full-stack web application
        acme/microservice - Microservice template
    
    Template: acme/api-starter
    
    ✓ Project created!
    ```

    <Tip>
      Choose a template that matches your use case (API, web app, microservices, etc.)
    </Tip>
    ```bash
    cd my-first-app
    
    # Review the generated project structure
    ls -la
    ```

    ```bash
    # Example project structure:
    my-first-app/
    ├── nitric.yaml          # Project configuration
    ├── package.json         # Dependencies (Node.js)
    ├── services/
    │   └── api.ts          # Your API service code
    └── README.md           # Project documentation
    ```

    ```bash
    # Install dependencies
    npm install  # or pip install -r requirements.txt, go mod tidy
    ```
  </Accordion>
  <Accordion title="Design" icon="pen-to-square">
    **Configure or review your application architecture:**

    ```bash
    # Login to platform
    nitric login
    
    # Open visual editor to design resources
    nitric edit
    ```

    <Note>
      Design your APIs, databases, storage, and other cloud resources
    </Note>
  </Accordion>
  <Accordion title="Develop" icon="code">
    **Build and test your application locally:**

    ```bash
    # Start development server with hot reload
    nitric dev
    ```

    ```bash
    ⚡ Nitric v1.0.0
       - App: my-first-app
       - Addr: http://localhost:50051
       - Dashboard: https://app.nitric.io/dev
    
    Services
    
    ✓ Starting [api]
    
    Entrypoints
    
    ✓ Starting [main] http://localhost:3000
    
    Use Ctrl-C to exit
    ```

    ```bash
    # Test your API
    curl http://localhost:3000/hello
    # {"message": "Hello World"}
    ```

    ```bash
    # Generate type-safe client SDKs for your APIs
    nitric generate --ts --ts-out ./sdk/typescript
    ```

    <Tip>
      Services automatically restart when you change code files
    </Tip>
  </Accordion>
  <Accordion title="Deploy" icon="cloud">
    **Configure deployment target and deploy to production:**

    Use the visual editor to choose from available targets:

    ```yaml
    nitric edit
    ```

    Or add deployment targets to `nitric.yaml`:

    ```bash
    targets:
      - nitric/aws@latest # or nitric/gcp@latest, nitric/azure@latest
    ```

    **Setup cloud credentials:**

    ```bash
    # AWS
    aws configure
    
    # Google Cloud
    gcloud auth application-default login
    
    # Azure
    az login
    ```

    **Generate Terraform:**

    ```bash
    nitric build
    ```

    **Setup cloud credentials:**

    ```bash
    # AWS
    aws configure
    
    # Google Cloud
    gcloud auth application-default login
    
    # Azure
    az login
    ```

    **Deploy with Terraform:**

    ```bash
    cd .nitric/stacks/my-first-app-aws-xyz
    terraform init -upgrade
    terraform plan
    terraform apply
    ```

    <Note>
      Requires cloud provider account and credentials configured
    </Note>
  </Accordion>
</AccordionGroup>

**What you just accomplished:**

- ✅ Correctly set up nitric
- ✅ A sample app running locally with hot reloads
- ✅ Generated Terraform deployment scripts
- ✅ Deployed to your chosen cloud provider (with proper credentials)

### `nitric new`

Create a new Nitric project from scratch or from a template.

```bash
nitric new my-app
```

```bash
Welcome to Nitric, this command will help you create a project from a template.

Project name: my-app

? Template: 
  > acme/api-starter - REST API template
    acme/web-app - Full-stack web application
    acme/microservice - Microservice template

Template: acme/api-starter

✓ Project created!

Next steps:
1. Run cd ./my-app to move to the project directory
2. Run nitric edit to start the Nitric editor
3. Design your app's resources and deployment targets
4. Run nitric dev to start the development server
5. Run nitric build to build the project for a specific platform

For more information, see the Nitric docs at https://nitric.io/docs
```

<AccordionGroup>
  <Accordion title="Flags" icon="gear">
    - `-f, --force` — Overwrite existing directory
  </Accordion>
</AccordionGroup>

### `nitric init`

Add Nitric to an existing application.

```bash
nitric init
```

```bash
Welcome to Nitric, this command will walk you through creating a nitric.yaml file.
This file is used to define your app's infrastructure, resources and deployment targets.

Here we'll only cover the basics, use nitric edit to continue editing the project.

Project name: my-project
Project description: My sample project

✓ Project initialized!
   Nitric project written to nitric.yaml

Next steps:
1. Run nitric edit to start the Nitric editor
2. Design your app's resources and deployment targets
3. Optionally, use nitric generate to generate the client libraries for your app
4. Run nitric dev to start the development server
5. Run nitric build to build the project for a specific platform

For more information, see the Nitric docs at https://nitric.io/docs
```

### `nitric build`

Generate Terraform infrastructure code for your deployment target.

First, define targets in your `nitric.yaml`:

```yaml
targets:
  - nitric/aws@latest # AWS provider
  - nitric/gcp@latest # Google Cloud provider
  - nitric/azure@latest # Azure provider
```

<AccordionGroup>
  <Accordion title="Single target" icon="check">
    ```bash
    nitric build
    ```

    ```bash
    ✓ Terraform generated successfully
      output written to ./.nitric/stacks/my-app-aws-12345
    
    Next steps:
    1. Run cd ./.nitric/stacks/my-app-aws-12345 to move to the stack directory
    2. Initialize the stack terraform init -upgrade
    3. Optionally, preview with terraform plan
    4. Deploy with terraform apply
    ```
  </Accordion>
  <Accordion title="Multiple targets" icon="list-check">
    ```bash
    nitric build
    ```

    ```bash
    ? Select a build target:
      > nitric/aws@latest
        nitric/gcp@latest
        nitric/azure@latest
    
    ✓ Terraform generated successfully
      output written to ./.nitric/stacks/my-app-aws-12345
    ```
  </Accordion>
  <Accordion title="No targets defined" icon="warning">
    ```bash
    nitric build
    ```

    ```bash
    No targets specified in nitric.yaml, run nitric edit to add a target
    ```
  </Accordion>
</AccordionGroup>

<Note>
  **Prerequisites for deployment**: Cloud provider account, Terraform CLI, and cloud provider CLI (AWS CLI, gcloud, or az CLI)
</Note>

## Environment Variables

Configure the CLI behavior with environment variables:

| Variable        | Description                                          | Default                    |
| --------------- | ---------------------------------------------------- | -------------------------- |
| `NITRIC_HOST`   | API server host                                      | `localhost`                |
| `NITRIC_PORT`   | API server port                                      | Auto-allocated 50051-50999 |
| `LOG_LEVEL`     | Logging verbosity (`debug`, `info`, `warn`, `error`) | `info`                     |
| `NITRIC_CONFIG` | Custom config file path                              | `~/.nitric/config.yaml`    |
| `NITRIC_TOKEN`  | Authentication token for CI/CD                       | -                          |

**Example:**

```bash
LOG_LEVEL=debug nitric dev
```

    ```bash
    # Nitric will automatically find next available port in 3000-3999 range

    ```

    <Tip>
      Nitric automatically handles port conflicts by finding the next available port
    </Tip>
  </Accordion>
  <Accordion title="Template Not Found" icon="image-slash">

    # Login to access team templates
    nitric login
    
    # Clear template cache
    rm -rf ~/.nitric/templates/
    ```
  </Accordion>
  <Accordion title="Build Failures" icon="hammer">

    LOG_LEVEL=debug nitric build

    rm -rf .nitric/
    nitric build
    ```
  </Accordion>
