## Get started with Suga

Deploy your first application on the Suga platform in just a few steps.

### Step 1: Install Suga CLI

<AccordionGroup>
  <Accordion icon="download" title="Install the Suga CLI">
    Download the Suga CLI for your operating system:
    
    **macOS:**
    ```bash
    brew install nitrictech/tap/suga
    ```
    
    **Windows:**
    ```powershell
    scoop bucket add nitrictech https://github.com/nitrictech/scoop-bucket
    scoop install suga
    ```
    
    **Linux:**
    ```bash
    curl -sSL https://suga.io/install.sh | sh
    ```
    
    Or download directly from [GitHub Releases](https://github.com/nitrictech/suga/releases)
    
    <Tip>Verify installation with `suga version`</Tip>
  </Accordion>
  <Accordion icon="user" title="Login to Suga">
    Authenticate with your Suga account:
    ```bash
    suga login
    ```
    
    This will open your browser to complete authentication.
    
    <Note>Don't have an account? Sign up at [app.suga.io](https://app.suga.io)</Note>
  </Accordion>
</AccordionGroup>

### Step 2: Create your first project

<AccordionGroup>
  <Accordion icon="folder" title="Create a new project">
    Use the `new` command to create a project from a template:
    ```bash
    suga new my-app
    ```
    
    You'll be prompted to choose a template. To see all available templates:
    ```bash
    suga templates
    ```
    
    <Tip>Choose a template that matches your use case (API, web app, microservices, etc.)</Tip>
  </Accordion>
  <Accordion icon="code" title="Explore the project structure">
    ```bash
    cd my-app
    ```
    
    Your project will include:
    - `suga.yaml` - Main configuration file
    - Service code in your chosen language
    - Infrastructure definitions
    - Environment configuration
    
    <Note>Project structure varies by template and language</Note>
  </Accordion>
</AccordionGroup>

### Step 3: Develop and deploy

<AccordionGroup>
  <Accordion icon="play" title="Run in development mode">
    Start your application locally:
    ```bash
    suga dev
    ```
    
    This will:
    - Start local development server
    - Enable hot reloading
    - Provide local endpoints for testing
    
    <Tip>Use `suga edit` to open the visual application editor</Tip>
  </Accordion>
  <Accordion icon="hammer" title="Build for deployment">
    Generate Terraform modules for deployment:
    ```bash
    suga build
    ```
    
    The build process:
    1. Validates your configuration
    2. Compiles your services
    3. Generates Terraform modules using Suga plugins or custom plugins
    4. Outputs infrastructure-as-code ready for deployment
    
    <Note>The generated Terraform modules can be deployed using standard Terraform workflows</Note>
  </Accordion>
</AccordionGroup>
