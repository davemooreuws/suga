## Get started with Suga in minutes

Deploy your first application on the Suga platform in just a few steps.

<Note>
  **Before you begin:** Make sure you have Suga CLI installed. See our [Installation Guide](/docs/cli/installation) for platform-specific instructions.
</Note>

<AccordionGroup>
  <Accordion title="Setup" defaultOpen icon="rocket">
    **Create and initialize your project:**

    ```bash
    # Create new project from template
    nitric new my-first-app
    ```

    ```bash
    Welcome to Nitric, this command will help you create a project from a template.
    
    Project name: my-first-app
    
    ? Template: 
      > acme/api-starter - REST API template
        acme/web-app - Full-stack web application
        acme/microservice - Microservice template
    
    Template: acme/api-starter
    
    ✓ Project created!
    ```

    <Tip>
      Choose a template that matches your use case or preferred framework.
    </Tip>
    ```bash
    cd my-first-app
    
    # Review the generated project structure
    ls -la
    ```

    ```bash
    # Example project structure:
    my-first-app/
    ├── nitric.yaml          # Project configuration
    ├── package.json         # Dependencies (Node.js)
    ├── services/
    │   └── api.ts          # Your API service code
    └── README.md           # Project documentation
    ```

    ```bash
    # Install dependencies
    npm install  # or pip install -r requirements.txt, go mod tidy
    ```
  </Accordion>
  <Accordion title="Design" icon="pen-to-square">
    **Configure or review your application architecture:**

    ```bash
    # Login to platform
    nitric login
    
    # Open visual editor to design resources
    nitric edit
    ```

    ![review.png](/docs/nitricedit.png)

    <Note>
      Design your APIs, databases, storage, and other cloud resources
    </Note>
  </Accordion>
  <Accordion title="Develop" icon="code">
    **Build and test your application locally:**

    ```bash
    # Start development server with hot reload
    nitric dev
    ```

    ```bash
    ⚡ Nitric v1.0.0
       - App: my-first-app
       - Addr: http://localhost:50051
       - Dashboard: https://app.nitric.io/dev
    
    Services
    
    ✓ Starting [api]
    
    Entrypoints
    
    ✓ Starting [main] http://localhost:3000
    
    Use Ctrl-C to exit
    ```

    ```bash
    # Test your API
    curl http://localhost:3000/hello
    # {"message": "Hello World"}
    ```

    <Tip>
      Services automatically restart when you change code files
    </Tip>
  </Accordion>
  <Accordion title="Deploy" icon="cloud">
    **Configure deployment target and deploy to production:**

    Use the visual editor to choose from available targets:

    ```bash
    nitric edit
    ```

    Or add deployment targets to `nitric.yaml`:

    ```bash nitric.yaml
    targets:
      - nitric/aws@latest # or nitric/gcp@latest, nitric/azure@latest
    ```

    **Setup cloud credentials:**

    ```bash
    # AWS
    aws configure
    
    # Google Cloud
    gcloud auth application-default login
    
    # Azure
    az login
    ```

    **Generate Terraform:**

    ```bash
    nitric build
    ```

    **Setup cloud credentials:**

    ```bash
    # AWS
    aws configure
    
    # Google Cloud
    gcloud auth application-default login
    
    # Azure
    az login
    ```

    **Deploy with Terraform:**

    ```bash
    cd .nitric/stacks/my-first-app-aws-xyz
    terraform init -upgrade
    terraform plan
    terraform apply
    ```

    <Note>
      Requires cloud provider account and credentials configured
    </Note>
  </Accordion>
</AccordionGroup>

**What you just accomplished:**

- ✅ Correctly set up Suga
- ✅ A sample app running locally with hot reloads
- ✅ Generated Terraform deployment scripts
- ✅ Deployed to your chosen cloud provider (with proper credentials)
