# Nitric CLI

The Nitric CLI provides everything you need to build, test, and deploy cloud-native applications.

## Quick Start

Get a working API running locally in under 5 minutes.

### Prerequisites

- [Nitric CLI installed](#installation)
- Docker (for local development)
- Node.js 18+, Python 3.9+, or Go 1.21+

**For cloud deployment (optional):**

- [Terraform CLI](https://terraform.io/downloads)
- Cloud provider CLI:
  - [AWS CLI](https://aws.amazon.com/cli/) for AWS deployments
  - [Google Cloud CLI](https://cloud.google.com/sdk/gcloud) for GCP deployments
  - [Azure CLI](https://docs.microsoft.com/en-us/cli/azure/) for Azure deployments

### Getting Started

<AccordionGroup>
  <Accordion icon="folder" title="1. Create Your Project">
    ```bash
    nitric new my-first-app
    ```
    
    Select a template when prompted.
    
    <Tip>Use `nitric templates` to see all available templates</Tip>
  </Accordion>
  
  <Accordion icon="download" title="2. Install Dependencies">
    ```bash
    cd my-first-app

    # For Node.js/TypeScript projects
    npm install

    # For Python projects
    pip install -r requirements.txt

    # For Go projects
    go mod tidy
    ```
  </Accordion>
  
  <Accordion icon="eye" title="3. Review Your Project">
    ```bash
    nitric edit
    ```
    
    Open the visual configuration editor to explore your project's resources and structure.
    
    **Alternatively**, you can review the `nitric.yaml` file directly to see your application configuration:
    
    ```bash
    cat nitric.yaml
    ```
    
    <Note>Project structure varies by template and language</Note>
  </Accordion>
</AccordionGroup>

### Development and Testing

<AccordionGroup>
  <Accordion icon="play" title="4. Start Development">
    ```bash
    nitric dev
    ```
    
    Your app is now running at `http://localhost:3000` (or next available port 3000-3999) with hot reload enabled.
    
    <Tip>Use `nitric edit` to open the visual configuration editor</Tip>
  </Accordion>
  
  <Accordion icon="code" title="5. Test Your API">
    ```bash
    # Test the hello endpoint
    curl http://localhost:3000/hello
    # {"message": "Hello World"}
    ```
  </Accordion>
  
  <Accordion icon="hammer" title="6. Make Changes (Hot Reload)">
    Edit your code and see changes instantly:
    
    ```typescript
    // services/api.ts
    api.get("/hello", async (ctx) => {
      return ctx.res.json({
        message: "Hello from Nitric! ðŸš€",
      });
    });
    ```
    
    Save the file â€” your changes are live immediately.
    
    <Note>Hot reload works automatically - no manual restarts needed</Note>
  </Accordion>
</AccordionGroup>

### Production Deployment

<AccordionGroup>
  <Accordion icon="gear" title="7. Configure Deployment Target">
    First, add a target to your `nitric.yaml`:
    
    ```yaml
    # nitric.yaml
    targets:
      - nitric/aws@latest # or nitric/gcp@latest, nitric/azure@latest
    ```
  </Accordion>
  
  <Accordion icon="hammer" title="8. Build for Production">
    ```bash
    nitric build
    ```
    
    This generates Terraform files in `.nitric/stacks/my-first-app-aws-xyz/`
    
    <Note>The generated Terraform files can be deployed using standard Terraform workflows</Note>
  </Accordion>
  
  <Accordion icon="cloud" title="9. Setup Cloud Credentials">
    Before deploying, configure credentials for your chosen provider:
    
    **AWS:**
    ```bash
    # Option 1: AWS CLI
    aws configure

    # Option 2: Environment variables
    export AWS_ACCESS_KEY_ID=your-access-key
    export AWS_SECRET_ACCESS_KEY=your-secret-key
    export AWS_DEFAULT_REGION=us-east-1
    ```
    
    **Google Cloud:**
    ```bash
    gcloud auth application-default login
    ```
    
    **Azure:**
    ```bash
    az login
    ```
  </Accordion>
  
  <Accordion icon="rocket" title="10. Deploy with Terraform">
    ```bash
    # Navigate to the generated stack
    cd .nitric/stacks/my-first-app-aws-xyz

    # Initialize Terraform
    terraform init -upgrade

    # Preview what will be created
    terraform plan

    # Deploy to your cloud provider
    terraform apply
    ```
    
    <Tip>Use `terraform plan` to preview changes before applying</Tip>
  </Accordion>
</AccordionGroup>

**What you just accomplished:**

- âœ… Full-stack app running locally
- âœ… Hot reload development
- âœ… Generated Terraform infrastructure code
- âœ… Deployed to your chosen cloud provider (with proper credentials)


### `nitric build`

Generate Terraform infrastructure code for your deployment target.

```bash
nitric build
```

**What actually happens**:

<!-- Source: cli/pkg/app/nitric.go:455-488 - Target selection and build -->

1. Checks `targets` in your `nitric.yaml`
2. If multiple targets, prompts you to choose one
3. Generates Terraform for the selected target
4. Outputs to a stack directory

**First, define targets in nitric.yaml:**

```yaml
targets:
  - nitric/aws@latest # AWS provider
  - nitric/gcp@latest # Google Cloud provider
  - nitric/azure@latest # Azure provider
```

<AccordionGroup>
  <Accordion icon="check" title="Example with single target">
    ```bash
    nitric build
    # âœ“ Terraform generated successfully
    #   output written to ./.nitric/stacks/my-app-aws-12345
    ```
  </Accordion>
  
  <Accordion icon="list" title="Example with multiple targets">
    ```bash
    nitric build
    # ? Select a build target:
    #   > nitric/aws@latest
    #     nitric/gcp@latest
    #     nitric/azure@latest
    # âœ“ Terraform generated successfully
    #   output written to ./.nitric/stacks/my-app-aws-12345
    ```
  </Accordion>
  
  <Accordion icon="warning" title="If no targets are defined">
    ```bash
    nitric build
    # No targets specified in nitric.yaml, run nitric edit to add a target
    ```
  </Accordion>
</AccordionGroup>

**What you get**:

- Cloud-specific Terraform configuration files
- Ready-to-deploy stack in `.nitric/stacks/` directory
- Provider-specific resource definitions (AWS, GCP, or Azure)

**Prerequisites for deployment**:

- Cloud provider account (AWS, GCP, or Azure)
- Terraform CLI installed
- Cloud provider CLI installed and configured:
  - AWS CLI (`aws configure`) for AWS deployments
  - Google Cloud CLI (`gcloud auth application-default login`) for GCP
  - Azure CLI (`az login`) for Azure deployments

## Environment Variables

Configure the CLI behavior with environment variables:

| Variable        | Description                                          | Default                     |
| --------------- | ---------------------------------------------------- | --------------------------- |
| `NITRIC_HOST`   | API server host                                      | `localhost`                 |
| `NITRIC_PORT`   | API server port                                      | Auto-allocated 50051-50999 |
| `LOG_LEVEL`     | Logging verbosity (`debug`, `info`, `warn`, `error`) | `info`                      |
| `NITRIC_CONFIG` | Custom config file path                              | `~/.nitric/config.yaml`     |
| `NITRIC_TOKEN`  | Authentication token for CI/CD                       | -                           |

**Example:**

```bash
LOG_LEVEL=debug nitric dev
```

  <Accordion icon="plus" title="Adding to Existing Project">
    ```bash

    nitric init

    # 2. Add cloud resources to your code
    # import { api, bucket } from '@nitric/sdk'

    # 3. Run locally

    # 4. Generate client SDKs
    nitric generate --ts --ts-out ./client
    ```
  </Accordion>
  
  <Accordion icon="users" title="Team Collaboration">
    ```bash
    # 1. Login to platform
    nitric login

    # 2. View team templates
    nitric templates

    # 3. Create from team template
    nitric new payment-service
    ```
  </Accordion>
  
  <Accordion icon="rocket" title="CI/CD Pipeline">
    ```bash
    #!/bin/bash
    # deploy.sh

    # Authenticate with Nitric (if needed)
    export NITRIC_TOKEN=$CI_NITRIC_TOKEN

    # Generate Terraform infrastructure
    nitric build

    # Generate SDKs
    nitric generate \
      --ts --ts-out ./web/sdk \
      --python --python-out ./api/sdk

    # Deploy with Terraform
    cd .nitric/stacks/*/  # Navigate to generated stack

    # Setup cloud credentials (AWS example)
    export AWS_ACCESS_KEY_ID=$CI_AWS_ACCESS_KEY_ID
    export AWS_SECRET_ACCESS_KEY=$CI_AWS_SECRET_ACCESS_KEY
    export AWS_DEFAULT_REGION=us-east-1

    # Initialize and deploy
    terraform init -upgrade
    terraform plan
    terraform apply -auto-approve
    ```
  </Accordion>
</AccordionGroup>
